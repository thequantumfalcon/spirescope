Getting Started

Install SpireScope and start the dashboard:

pip install -e .
spirescope

Or run directly with python -m sts2. Your browser will open automatically at http://127.0.0.1:8000.

Save File Setup

SpireScope auto-detects your STS2 save directory. If it can't find it, set the STS2_SAVE_DIR environment variable:

# Windows
set STS2_SAVE_DIR=%APPDATA%\SlayTheSpire2\steam\YOUR_ID\profile1\saves

# macOS / Linux
export STS2_SAVE_DIR=~/.local/share/SlayTheSpire2/steam/YOUR_ID/profile1/saves

Replace YOUR_ID with your Steam ID. Check the home page's "Data Sources" section to verify the connection.

Updating Data

SpireScope ships with baked-in game data. To fetch the latest cards, relics, and potions from the wiki:

spirescope update

If you don't have internet access or just want to discover new entities from your save files:

spirescope update --save-only

You can also hot-reload data without restarting by calling POST /api/reload with header X-Admin-Token: YOUR_TOKEN. Set SPIRESCOPE_ADMIN_TOKEN env var or enable debug logging to see the auto-generated token.

Page Tour

Cards — Browse all cards with filters by character, type, rarity, cost, and keyword. Click any card for synergies, stats, and community tips.

Relics — Filter by character and rarity. Detail pages show runs where each relic appeared.

Potions — All potions filterable by rarity.

Enemies — Filter by act and type (normal/elite/boss). Detail pages show attack patterns, strategy tips, and your personal win/loss stats.

Events — Every event with optimal choices and outcomes, filterable by act.

Strategy Guides — Per-character archetypes, key cards, key relics, and general tips. Access from character cards on the home page.

Run History — All completed runs with floor-by-floor breakdown, HP tracking, card picks, and damage taken.

Analytics — Aggregate statistics across all your runs: win rates, character performance, card pick rates, and more.

Community — Community tier lists and meta discussion from Reddit.

Live Run Tracking

The Live Run page updates in real time via Server-Sent Events (SSE) — no page reloads needed. It shows your current deck, relics, potions, HP, floor history, and a live deck analysis.

For co-op, use the player selector or add ?player=1 to track a different player in the same run.

The SSE stream checks for changes every 3 seconds and auto-disconnects after 5 minutes of inactivity.

Deck Analyzer

Select cards from the full card pool to simulate a deck. The analyzer provides:

  • Archetype detection (matches against known strategy archetypes)
  • Type breakdown (attacks, skills, powers)
  • Cost curve visualization
  • Keyword frequency analysis
  • Weakness identification and suggestions

Decks are saved to your browser's localStorage so you can return to them later.

Analytics

The analytics page aggregates data across all your completed runs:

  • Overall and per-character win rates
  • Floor survival curves
  • Most picked and highest win-rate cards
  • Common causes of death

Analytics are cached for 60 seconds and recompute automatically when new runs are detected.

Community Data

Scrape community tips and meta posts from Reddit:

spirescope community

Community tips appear on card, relic, and enemy detail pages. Meta posts appear on the Community page.

Mod Support

SpireScope can load custom mod data (cards, relics, enemies) from JSON files. Place .json files in the mods directory:

# Exe distribution: mods/ folder next to spirescope.exe
# Development: sts2/data/mods/

You can also set STS2_MODS_DIR to a custom path. Each JSON file should follow this format:

{
  "mod_name": "My Mod",
  "cards": [
    {"id": "MOD.CARD_NAME", "name": "Card Name", "character": "Ironclad",
     "cost": "1", "type": "Attack", "rarity": "Common", "keywords": ["Block"]}
  ],
  "relics": [
    {"id": "MOD.RELIC_NAME", "name": "Relic Name", "rarity": "Common",
     "description": "Does something cool."}
  ],
  "enemies": [
    {"id": "MOD.ENEMY_NAME", "name": "Enemy Name", "act": ["1"],
     "type": "normal", "hp_range": "40-50", "patterns": [], "tips": []}
  ]
}

All entity types are optional per file. Base game data always takes priority — if a mod ID conflicts with a base card, the mod entry is skipped. Mod entities are tagged with a Mod badge. After adding mod files, use POST /api/reload or restart to load them.

Troubleshooting

"No save files found" — Set STS2_SAVE_DIR to point to your saves directory. Make sure you've launched STS2 at least once.

Empty card/relic lists — Run spirescope update to fetch data from the wiki.

Live run not updating — The game must be running with an active run. Check that your save directory is correct.

Port already in use — Set STS2_PORT=8001 (or any free port) before starting.

Rate limited (429) — The server limits to 60 requests per minute per IP. Wait a moment and try again.
