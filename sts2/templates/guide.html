Getting Started

Install SpireScope and start the dashboard:

pip install -e .
spirescope

Or run directly with python -m sts2. Source installs open your browser automatically at http://127.0.0.1:8000. The packaged Windows build keeps a console window open and serves the same local address by default.

Save File Setup

SpireScope auto-detects your STS2 save directory. If it can't find it, set the STS2_SAVE_DIR environment variable:

# Windows
set STS2_SAVE_DIR=%APPDATA%\SlayTheSpire2\steam\YOUR_ID\profile1\saves

# macOS / Linux
export STS2_SAVE_DIR=~/.local/share/SlayTheSpire2/steam/YOUR_ID/profile1/saves

Replace YOUR_ID with your Steam ID. Check the home page's "Data Sources" section to verify the connection.

Updating Data

SpireScope ships with baked-in game data. To fetch the latest cards, relics, and potions from the wiki:

spirescope update

If you don't have internet access or just want to discover new entities from your save files:

spirescope update --save-only

You can also hot-reload data without restarting by calling POST /api/reload with header X-Admin-Token: YOUR_TOKEN. Set SPIRESCOPE_ADMIN_TOKEN env var or enable debug logging to see the auto-generated token.

Page Tour

Cards — Browse all cards with filters by character, type, rarity, cost, and keyword. Click any card for synergies, stats, and community tips.

Relics — Filter by character and rarity. Detail pages show runs where each relic appeared.

Potions — All potions filterable by rarity.

Enemies — Filter by act and type (normal/elite/boss). Detail pages show attack patterns, strategy tips, and your personal win/loss stats.

Events — Every event with optimal choices and outcomes, filterable by act.

Strategy Guides — Per-character archetypes, key cards, key relics, and general tips. Access from character cards on the home page.

Run History — All completed runs with floor-by-floor breakdown, HP timeline chart, card picks, cards offered (what you rejected), potions gained, monsters fought, gold per floor, and damage taken. Filter by character, result, ascension, game version, or time range. Export as JSON or standalone HTML. Select two runs to compare side-by-side.

Analytics — Aggregate statistics across all your runs: win rates, character performance, card pick rates, boss matchups, per-character relic tiers, card pick heatmap, and more. Filter by ascension level, game version, or time range.

Records — Personal records: fastest win, highest ascension, best streak, biggest/smallest deck, flawless bosses, per-character breakdowns.

Community — Community tier lists and meta discussion from Reddit and Steam.

Live Run Tracking

The Live Run page updates in real time via Server-Sent Events (SSE) — no page reloads needed. It shows your current deck, relics, potions, HP, floor history, encounters won, events encountered, and a live deck analysis with archetype detection and coaching alerts.

For co-op, use the player selector or add ?player=1 to track a different player in the same run.

The SSE stream checks for changes every 3 seconds and auto-disconnects after 5 minutes of inactivity.

Deck Analyzer

Select cards from the full card pool to simulate a deck. The analyzer provides:

  • Archetype detection (matches against known strategy archetypes)
  • Type breakdown (attacks, skills, powers)
  • Cost curve visualization
  • Keyword frequency analysis
  • Weakness identification and suggestions

Decks are saved to your browser's localStorage so you can return to them later.

Analytics

The analytics page aggregates data across all your completed runs:

  • Overall and per-character win rates
  • Floor survival curves and HP tracking
  • Most picked and highest win-rate cards with pick heatmap
  • Boss matchups: win rate and avg damage per boss per character
  • Relic tier list per character
  • Per-act breakdown, combat efficiency, encounter danger ratings
  • Gold economy, healing sources, co-op stats
  • Card regret analysis and archetype performance

Use the filter bars to focus by ascension level, game version, or time range (7d/30d/90d presets or custom dates).

Analytics are cached for 60 seconds and recompute automatically when new runs are detected.

Community Data

Fetch community tips and meta posts from Reddit and Steam:

spirescope community

Community tips appear on card, relic, and enemy detail pages. Meta posts appear on the Community page.

Mod Support

SpireScope can load custom mod data (cards, relics, enemies) from JSON files. Place .json files in the mods directory:

# Exe distribution: mods/ folder next to spirescope.exe
# Development: sts2/data/mods/

You can also set STS2_MODS_DIR to a custom path. Each JSON file should follow this format:

{
  "mod_name": "My Mod",
  "cards": [
    {"id": "MOD.CARD_NAME", "name": "Card Name", "character": "Ironclad",
     "cost": "1", "type": "Attack", "rarity": "Common", "keywords": ["Block"]}
  ],
  "relics": [
    {"id": "MOD.RELIC_NAME", "name": "Relic Name", "rarity": "Common",
     "description": "Does something cool."}
  ],
  "enemies": [
    {"id": "MOD.ENEMY_NAME", "name": "Enemy Name", "act": ["1"],
     "type": "normal", "hp_range": "40-50", "patterns": [], "tips": []}
  ]
}

All entity types are optional per file. Base game data always takes priority — if a mod ID conflicts with a base card, the mod entry is skipped. Mod entities are tagged with a Mod badge. After adding mod files, use POST /api/reload or restart to load them.

Advanced Analytics

SpireScope includes advanced analytics that go deeper than any STS2 companion tool. Here's how to use each one:

The Graveyard

Visit /graveyard to see a memorial wall for every dead run. Each gravestone shows a procedural epitaph based on what actually happened — potion hoarding, oversized decks, instant deaths, marathon losses. Screenshot and share your favorites.

Ghost Run Comparison

During a live run, SpireScope finds your best comparable historical run and shows floor-by-floor deltas. Green arrows mean you're ahead of your personal best. Red arrows mean you're behind. Works automatically — just open the live tracker.

Tilt Detection

SpireScope tracks your session momentum. If your recent runs are getting shorter and your floor average is dropping, a gentle warning appears on the home page suggesting a break. No judgment — just data.

Anti-Pattern Detection

After enough runs, SpireScope identifies recurring habits that hurt your win rate:

  • The Hoarder — dying with unused potions
  • The Greedy Builder — losing decks are bigger than winning decks
  • The Coward — skipping too many elites
  • Potion Paralysis — consistently dying with potions in hand

Shown on the analytics page once you have 5+ runs.

Deck Health Score

The deck analyzer computes a synergy graph where cards are connected by shared keywords. The health score (0-100) measures how well your cards work together. Orphan cards — those with zero connections — are highlighted. Available on the deck analyzer and live tracker.

Autopsy Report

When a run ends in death, open the run detail page to see the Autopsy Report. Five diagnostic agents analyze your run from different angles — deck coherence, combat tempo, economy, adaptation, and (uniquely) what went right. The causal chain traces the sequence of decisions that led to death.

Cascade Map

On any completed run's detail page, click a card to see its downstream impact. SpireScope compares your performance before and after picking that card — damage taken, fight length, floors survived. One card, its entire ripple through your run.

Prophecy Engine

Before starting a run, the home page shows a prediction based on your history with that character: win probability, danger zone floors, and a recommendation. After the run, the prophecy is graded. Requires 5+ runs with the same character.

Hypothesis Lab

Test your strategic beliefs with data. Register a hypothesis like "skipping elites helps" and SpireScope tracks qualifying runs, updates Bayesian probabilities, and delivers a verdict: confirmed, refuted, or inconclusive.

Rivalry Seeds

Export a run as JSON from the run detail page. Share the seed with a friend. They play the same seed and export their run. Import both on the rivalry page to see a floor-by-floor decision diff — where you picked differently and who survived longer.

Run Integrity

Every run gets a SHA-256 hash chain (Merkle root) that proves the run data is unmodified. Shown on the run detail page. Share your hash to verify achievements without trusting a central server.

Keyboard Shortcuts

Press ? anywhere to see the shortcut overlay. Available shortcuts:

  • / — Focus search (or deck search on the Deck Analyzer page)
  • h — Home
  • c — Cards
  • r — Relics
  • a — Analytics
  • d — Deck Analyzer
  • l — Live Run
  • Esc — Close overlay

Shortcuts are disabled when typing in input fields.

Troubleshooting

"No save files found" — Set STS2_SAVE_DIR to point to your saves directory. Make sure you've launched STS2 at least once.

Empty card/relic lists — Run spirescope update to fetch data from the wiki.

Live run not updating — The game must be running with an active run. Check that your save directory is correct.

Port already in use — Set STS2_PORT=8001 (or any free port) before starting.

Rate limited (429) — The server limits to 60 requests per minute per IP. Wait a moment and try again.

Windows warns about the download — The packaged build is unsigned, so SmartScreen or mod-host scanners may still show a reputation warning. The release zip ships with a .sha256 file so you can verify integrity, and the app runs locally on 127.0.0.1 by default.
